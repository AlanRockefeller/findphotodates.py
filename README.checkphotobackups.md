# checkphotobackups.py

A safety tool for verifying that files in a specific directory (e.g., a laptop folder, SD card, or staging area) are already backed up on your external drives.

It compares a **Target Directory** against a set of **Inventory Files** (generated by Alan's `findphotodates.py`) to confirm if every single file exists somewhere else.

## Features

- **Dual Format Support:** Works with both **New** (tab-separated) and **Old** (line-based) inventory file formats.
- **Smart Matching:** Uses a 4-tier matching system to verify files (Strongest to Weakest).
- **"Safe to Delete" Detection:** Automatically identifies subdirectories where **100%** of files have been confirmed as backed up.
- **Detailed Reporting:** Generates a CSV report of every file's status and a simple text list of missing files.

## Prerequisites

- **Python 3.7+**
- (Optional) **`exifread`** library. This improves accuracy for JPEGs by reading the internal "Date Taken" tag.

  ```bash
  pip install exifread
  ```

  *If not installed, the script works fine but will rely on file size and name matching.*

## Usage

### 1. Prepare Inventories

Ensure you have your inventory files ready. These are typically text files generated by `findphotodates.py` representing the contents of your external drives.

- *Default location:* The script looks for files matching `~/*:photo.taken.dates.txt`.

### 2. Run the Check

Run the script, pointing it to the folder you want to verify.

```bash
# Basic usage (uses default inventory location)
python3 checkphotobackups.py --target "/path/to/folder/to/check"
```

Example: Checking an import folder on your laptop against all known drive inventories:

```bash
python3 checkphotobackups.py --target "/Users/alan/Pictures/2023_Import_Staging"
```

### 3. Review the Output

The script prints a summary to the console and generates two files in your current directory:

- `backup_check_report.csv`: A detailed spreadsheet.
  - **Found:** `yes` (Safe) or `no` (Not safe).
  - **Found Drives:** Which drive(s) the file was found on.
  - **Match Strength:** How confident the match is (see below).
- `missing_from_backups.txt`: A clean list of file paths that were not found in any inventory.

## Matching Logic

The script attempts to match target files to inventory records using the following priority:

1. **Strong:** Matches Basename + File Size + Date Taken *(Requires NEW inventory format).*
2. **Weak:** Matches Basename + File Size *(Requires NEW inventory format).*
3. **Medium:** Matches Basename + Date Taken *(Works with OLD inventory format).*
4. **Name-only:** Matches Basename only *(Least safe; useful as a last resort).*

## Command Line Arguments

| Argument | Description | Default |
|---|---|---|
| `--target` | Required. The folder you want to verify. | N/A |
| `--inventories` | Comma-separated list of inventory files. | `~/*:photo.taken.dates.txt` |
| `--exts` | Comma-separated file extensions to scan (jpg, jpeg, png, tif, orf, nef, mov, mp4, etc.). | *(depends on script defaults)* |
| `--exclude-path-regex` | Regex to ignore specific paths in inventories (e.g., Trash). | `/$RECYCLE\.BIN/, /.Trash/, etc.` |
| `--out-csv` | Filename for the report CSV. | `backup_check_report.csv` |
| `--missing-list` | Filename for the missing files list. | `missing_from_backups.txt` |

## Interpreting "Safe to Delete"

At the end of the execution, the script analyzes per-directory statistics.

### "Directories safe to delete"

If a directory is listed here, it means every single file inside that specific folder was found in your backups.

Example output:

```text
=== Directories safe to delete (2) ===
  2023/London_Trip/  (145 files, all backed up)
  2023/Family_Picnic/  (88 files, all backed up)
```

### "Directories NOT fully backed up"

These folders contain at least one file that was not found in the inventories. Do not delete these folders yet.

Example output:

```text
=== Directories NOT fully backed up (1) ===
  2024/New_Photos/  (5 of 12 files missing from backups)
```
